## Orchestration вҖ” "Fund Event" (Per-Event Fee)

```
POST /events/{eid}/funding
  {funding_type: "per_event_fee"}
вҶ’ event funded, limits lifted to full access
```

## Orchestration вҖ” "Fund Event" (Subscription Slot)

```
POST /events/{eid}/funding
  {funding_type: "subscription_slot", slot_type: "one_off"}
вҶ’ slot consumed from user's membership quota, event funded
```

## Orchestration вҖ” "Spread Costs"

```
POST /events/{eid}/funding/cost-spread
  {spread_to: "all_participants"}
вҶ’ line item created in a system transaction, spreading $2
  across all event participants as consumption splits
```

## Orchestration вҖ” Page Load

```
GET /events/{eid}/funding
вҶ’ returns: funding_status, funding_type, cost_spread status
```

If unfunded, also needs membership quota info:

```
GET /memberships/me
вҶ’ returns: available slots for the subscription option display
```

## Smart Defaults

- Per-event fee pre-selected if user has no subscription slots available
- Subscription slot pre-selected if user has available slots
- Cost spread shown immediately after funding вҖ” natural next step
- Cost spread amount capped at per-event fee ($2) to prevent profiteering
- Slot type explanation is inline вҖ” users don't need to understand the quota model in depth
- Ongoing vs one-off guidance: "Use ongoing for events that will last a while (trips). Use one-off for single-day events (dinners)."

## Error States

| Error | Display |
|-------|---------|
| No slots available | "No subscription slots available. Use per-event fee or upgrade your plan." вҶ’ link to S13 |
| Already funded | "This event is already funded" (show funded state) |
| Cost spread already exists | "Cost spread already active" (prevent duplicate вҖ” known backend gap R9) |
| Event locked | "This event is locked and can no longer be modified" |
